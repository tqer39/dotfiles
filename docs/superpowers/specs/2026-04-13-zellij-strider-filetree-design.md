# Zellij default レイアウトに strider ファイルツリーを常駐表示する

## 概要

Zellij の default レイアウトの左ペインに `zellij:strider` プラグインを常駐させ、Windows エクスプローラーのようなファイルツリーを常時表示する。

## 背景

- Claude Code やターミナル作業中に、現在のパスやディレクトリ構造が把握しづらい
- 常にファイルツリーが見えることで、ナビゲーションの効率が上がる

## 設計

### 変更対象

`src/.config/zellij/layouts/default.kdl` の1ファイルのみ。

### レイアウト構造

```text
┌──────────────────────────────────────────┐
│  tab-bar (上部、1行)                      │
├──────────┬───────────────────────────────┤
│          │                               │
│  strider │  children (メインペイン)       │
│  25% 幅  │  75% 幅                       │
│  cwd="/" │                               │
│          │                               │
├──────────┴───────────────────────────────┤
│  status-bar (下部、2行)                   │
└──────────────────────────────────────────┘
```

### 具体的な変更

`default_tab_template`内の`children`を`split_direction="vertical"`のpaneで囲む。
左側にstrider(`size "25%"`、`cwd "/"`)、右側に`children`を配置する。

### 影響範囲

- `config.kdl`の変更不要(striderプラグイン定義済み)
- 既存のキーバインドへの影響なし
- `main`タブも`default_tab_template`を利用するため、そのまま適用される
- メインペインで`Ctrl+p f`を押せばフルスクリーンになり、striderを隠せる

## 不採用案

- **B. キーバインドでフローティング起動**: 常時表示の要件を満たさない
- **C. 常駐+フローティングのハイブリッド**: 過剰で紛らわしい
